================================================================================
  STAGE C — EDGE ADAPTIVE CHARGING (EACI)  (NPU — Hexagon)
================================================================================

  The EACI policy also runs on the NPU as a small ONNX model.
  It takes the ML output and decides the safe charging current.

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │   INPUT:                                                         │
  │    ML says: Risk=82.25, T1=119°C, SOH=92.12%                   │
  │                                                                  │
  │   EACI DECIDES:                                                  │
  │    • Temperature too high → reduce charging current             │
  │    • Risk too high → cut to 50% max                             │
  │    • SOH still good → no extra reduction needed                 │
  │                                                                  │
  │   OUTPUT:                                                        │
  │    • Safe charge current: 5.0A (reduced from 12A)               │
  │    • Mode: SAFE_DERATED                                         │
  │                                                                  │
  │   SENDS BACK TO ARDUINO UNO Q VIA MQTT:                         │
  │    Topic: ev/control/eaci                                       │
  │    Payload: {"max_charge_a": 5.0, "pwm_duty_pct": 41.7}        │
  │                                                                  │
  │   WHY NPU HERE:                                                  │
  │    • EACI must respond in <10ms (real-time safety)              │
  │    • NPU handles this in ~2ms                                   │
  │    • CPU would take 20-30ms (too slow for safety)               │
  │                                                                  │
  └──────────────────────────────────────────────────────────────────┘
