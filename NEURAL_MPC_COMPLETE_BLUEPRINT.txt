================================================================================
  PART 2 — THEORY (What Neural-MPC IS, step by step)
================================================================================

  ─────────────────────────────────────────────────────────────────────
  CONCEPT 1: What is a "Model" in MPC?
  ─────────────────────────────────────────────────────────────────────

  A "model" in control theory means:
  "Given the current state, I can PREDICT what happens next."

  Example:
    If Temperature = 80°C and Current = 10A and Pump = 40% speed...
    → Temperature in 2 seconds = 85°C  (it will RISE)

    If Temperature = 80°C and Current =  5A and Pump = 80% speed...
    → Temperature in 2 seconds = 76°C  (it will FALL)

  MPC uses this model to ASK:
  "What settings NOW will give me the BEST outcome in the next 10 seconds?"

  ─────────────────────────────────────────────────────────────────────
  CONCEPT 2: Why is traditional MPC slow?
  ─────────────────────────────────────────────────────────────────────

  Traditional MPC solves this at every 100ms step:

  MINIMIZE:  sum of (T_predicted - T_safe)²  +  (I_predicted - I_limit)²
  OVER:      1000 possible combinations of [PWM_charge, PWM_motor, PWM_cool]
  SUBJECT TO: V_cell > 2.5V, T1 < 60°C, I < 15A

  This requires solving quadratic programming matrices:

     [H] × u = -[f]     (where H is a 1000×1000 matrix)

  On standard CPU: takes 800ms to 3 seconds.
  On Snapdragon X NPU: impossible to run this math directly.

  ─────────────────────────────────────────────────────────────────────
  CONCEPT 3: What is Neural-MPC? (The Smart Shortcut)
  ─────────────────────────────────────────────────────────────────────

  Neural-MPC TRAINS a neural network to LEARN the answer pattern:

  TRAINING (done once on a PC, not on the device):
  ─────────────────────────────────────────────────
  Step A: Run traditional MPC solver 100,000 times
          with different sensor inputs.

  Step B: Record every input-output pair:
          Input:  [V1, V2, V3, V4, I, T1, T2, Vib, Gas, SOH, Risk]
          Output: [PWM_charge, PWM_motor, PWM_cool]

  Step C: Train a 3-layer neural network to map Input → Output.
          (This is a standard MLP — Multi-Layer Perceptron)

  Step D: Quantize the model to INT8 for Hexagon NPU.
          Export as ONNX file → "nmpc_controller.onnx"

  DEPLOYMENT (runs every 100ms on Snapdragon X):
  ───────────────────────────────────────────────
  Step E: Load "nmpc_controller.onnx" with QNN execution provider.
          NPU runs inference: 11 inputs → 3 outputs in < 2ms.
          No math solving. No matrix inversion. Pure pattern matching.

  ─────────────────────────────────────────────────────────────────────
  CONCEPT 4: What does the Neural Network look like inside?
  ─────────────────────────────────────────────────────────────────────

                      INPUT LAYER (11 neurons)
                      ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐
                      │V1│ │V2│ │V3│ │V4│ │I │ │T1│ │T2│ │Vb│ │Gs│ │SH│ │Rk│
                      └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘ └┬─┘
                       │    │    │    │    │    │    │    │    │    │    │
                       └────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘
                                              │
                                    HIDDEN LAYER 1 (64 neurons)
                                    [ReLU activation]
                                    Each neuron = weighted sum of all inputs
                                              │
                                    HIDDEN LAYER 2 (64 neurons)
                                    [ReLU activation]
                                              │
                                    HIDDEN LAYER 3 (32 neurons)
                                    [ReLU activation]
                                              │
                                     OUTPUT LAYER (3 neurons)
                                     [Sigmoid → scaled to 0-100%]
                                    ┌──────────────────────────┐
                                    │ PWM_charge  = 41.7%      │
                                    │ Motor_scale = 0.68       │
                                    │ Pump_PWM    = 88.0%      │
                                    └──────────────────────────┘

  Total parameters: ~12,000 weights.
  Model size: ~48 KB (fits easily in NPU cache).
  Inference: ~1.9ms on Hexagon NPU.


================================================================================
  PART 3 — SOFTWARE SIDE (What code runs where)
================================================================================

  ┌──────────────────────────────────────────────────────────────────────┐
  │  FILE 1: firmware/main.c  (runs on STM32 via Zephyr RTOS)           │
  │                                                                       │
  │  WHAT IT DOES:                                                        │
  │   1. Reads all 9 sensors every 100ms                                 │
  │   2. Writes sensor values to SRAM at 0x20000000                     │
  │   3. Reads PWM commands back from SRAM at 0x20000010                │
  │   4. Applies values to GPIO pins PB4, PB5, PB6                      │
  │                                                                       │
  │  KEY CODE IN firmware/main.c:                                         │
  │  ─────────────────────────────                                        │
  │  Reading NMPC commands from shared memory:                            │
  │                                                                       │
  │    uint8_t *nmpc = (uint8_t *)0x20000010;                           │
  │    uint8_t charge_pwm = nmpc[0];  // 0x6A = 41.7%                   │
  │    uint8_t motor_pwm  = nmpc[1];  // 0xAE = 68.0%                   │
  │    uint8_t cool_pwm   = nmpc[2];  // 0xE0 = 88.0%                   │
  │                                                                       │
  │  Applying to hardware:                                                │
  │                                                                       │
  │    pwm_set_duty(dev_mosfet,  charge_pwm);  // controls charging      │
  │    pwm_set_duty(dev_motor,   motor_pwm);   // caps motor torque      │
  │    pwm_set_duty(dev_cooling, cool_pwm);    // runs coolant pump      │
  └──────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────┐
  │  FILE 2: backend.py  (runs on Snapdragon X)                         │
  │                                                                       │
  │  WHAT IT DOES:                                                        │
  │   1. Receives MQTT telemetry from Arduino Q                          │
  │   2. Runs Model 1 (anomaly_model.onnx) on NPU → Risk Score           │
  │   3. Runs Model 2 (soh_model.onnx) on NPU → SOH %                   │
  │   4. Runs Model 3 (nmpc_controller.onnx) on NPU → 3 PWM values      │
  │   5. Publishes PWM commands back via MQTT to Arduino Q               │
  │   6. Broadcasts full trace to WebSocket dashboard                    │
  │                                                                       │
  │  NEW CODE TO ADD in backend.py:                                       │
  │  ─────────────────────────────                                        │
  │    # Load Neural-MPC model                                            │
  │    nmpc_sess, nmpc_inp = load_session("nmpc_controller.onnx","NMPC") │
  │                                                                       │
  │    def run_nmpc(vec, soh, risk) -> dict:                              │
  │        # Extend the vector with ML outputs                           │
  │        full_vec = np.append(vec, [soh/100.0, risk/100.0])           │
  │        full_vec = full_vec.reshape(1, -1).astype(np.float32)        │
  │        out = nmpc_sess.run(None, {nmpc_inp: full_vec})[0]           │
  │        return {                                                       │
  │            "charge_pwm": float(out[0] * 100),                       │
  │            "motor_scale": float(out[1]),                             │
  │            "cool_pwm":   float(out[2] * 100)                        │
  │        }                                                              │
  │                                                                       │
  │    # Send result back to STM32 via MQTT                              │
  │    client.publish("ev/control/nmpc", json.dumps(nmpc_result))        │
  └──────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────┐
  │  FILE 3: train_nmpc.py  (run ONCE on any PC to generate model)      │
  │                                                                       │
  │  WHAT IT DOES:                                                        │
  │   1. Generates 100,000 synthetic training scenarios                  │
  │   2. For each scenario, runs a traditional MPC solver                │
  │   3. Records the input-output pairs                                  │
  │   4. Trains a 3-layer MLP neural network on those pairs             │
  │   5. Exports to "nmpc_controller.onnx"                               │
  │                                                                       │
  │  KEY IDEA: The model does NOT need real driving data.                 │
  │  It learns from mathematically generated MPC optimization answers.   │
  └──────────────────────────────────────────────────────────────────────┘
