 STAGE 2 — ZEPHYR RTOS SCHEDULER     (STM32 MCU)
================================================================================

  ┌──────────────────────────────────────────────────────────────────┐
  │  TASK 1 — SENSOR READING              runs every 100ms           │
  │   • Read all 7 sensors (voltage, current, temp, gas, vibration)  │
  │   • Filter noise (5-point median filter)                         │
  │   • Calculate rates of change (dV/dt, dT/dt, dI/dt)              │
  │   • Write 35 parameters to SHARED MEMORY                         │
  │                                                                  │
  │  TASK 2 — DISPLAY MANAGER             runs every 50ms            │
  │   • Check if user touched TFT screen                             │
  │   • Switch page if touched                                       │
  │   • Update current page graphics                                 │
  │                                                                  │
  │  TASK 3 — SAFETY OUTPUT               runs every 100ms           │
  │   • Read ML result from shared memory                            │
  │   • Set charging PWM based on EACI output                        │
  │   • Update dot matrix display (Normal / Warning / Danger)        │
  └──────────────────────────────────────────────────────────────────┘
                                     │
                     35 parameters written to SHARED MEMORY (SRAM)
                                     ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │                 SHARED MEMORY (SRAM IPC BRIDGE)                  │
  │                                                                    │
  │   How STM32 and QRB2210 talk — like a shared whiteboard.          │
  │                                                                    │
  │        STM32 WRITES ───────────────► QRB2210 READS               │
  │        (sensor data, touch events)   (picks up data)             │
  │                                                                    │
  │        STM32 READS  ◄─────────────── QRB2210 WRITES              │
  │        (charging commands, ML result)(sends decisions)            │
  └──────────────────────────────────────────────────────────────────┘
                                     │
                       QRB2210 reads 35 parameters

 STAGE 8 — MQTT DATA PUBLISH         (QRB2210 sends to network)
================================================================================

  QRB2210 bundles everything into one message, published every 500ms.

  ┌──────────────────────────────────────────────────────────────────┐
  │  TOPIC:  ev/guardian/telemetry                                    │
  │                                                                    │
  │  CONTAINS:                                                        │
  │   • All cell voltages (C1, C2, C3, C4)                            │
  │   • Current, Pack Voltage, SoC                                    │
  │   • Temperatures (T1, T2, Delta_T)                                │
  │   • Gas sensor (CO_PPM)                                           │
  │   • Vibration (RMS, Peak, Freq)                                   │
  │   • ML Output (Fault, SOH, RUL, Risk)                             │
  │   • EACI Output (safe current, PWM, mode)                         │
  └──────────────────────────┬─────────────────────────────────────────┘

 HARDWARE ALLOCATION SUMMARY
================================================================================

  ┌──────────────────────────────────────────────────────────────────┐
  │   STM32 MCU (fast, real-time)   │   QRB2210 MPU (powerful)        │
  │   ───────────────────────────   │   ───────────────────────────  │
  │   Stage 1: Read all sensors     │   Stage 3: Run ML models        │
  │   Stage 2: RTOS task scheduling │   Stage 4: Generate output      │
  │   Stage 6: Drive TFT display    │   Stage 5: EACI decisions       │
  │   Stage 7: Drive dot matrix     │   Stage 8: MQTT publish         │
  │   + PWM output to MOSFET        │                                  │
  │                                  │                                  │
  │   WHY HERE:                      │   WHY HERE:                     │
  │    • Must be instant (<1ms)     │    • Needs heavy compute        │
  │    • Safety-critical            │    • Needs Python/ONNX          │
  │    • Direct hardware pins       │    • Needs network stack        │
  └──────────────────────────────────────────────────────────────────┘
