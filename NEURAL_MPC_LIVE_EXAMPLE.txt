================================================================================
  STEP 3 — NEURAL-MPC CONTROLLER RUNS (on NPU)         ← 2 ms
================================================================================

  The Neural-MPC model now receives ALL of this:

  ┌──────────────────────────────────────────────────────────────────────┐
  │  STATE VECTOR input to Neural-MPC:                                   │
  │                                                                       │
  │  Raw Sensors:   [2.959, 2.953, 2.958, 2.961,   ← 4 cell voltages     │
  │                  -12.257,                        ← total current      │
  │                  119.953, 97.321,                ← 2 temperatures     │
  │                  0.170, 0.524]                   ← vibration, gas     │
  │                                                                       │
  │  ML Outputs:    [SOH=92.12, Risk_Score=82.25]   ← from Model 1 & 2  │
  └──────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
         WHAT THE NPU DOES:
         Instead of solving quadratic optimization equations (which takes
         seconds), it multiplies this 11-dimensional vector through a
         trained deep neural network on the Hexagon tensor cores.
                              │
                              ▼
  ┌──────────────────────────────────────────────────────────────────────┐
  │  NEURAL-MPC OUTPUT  (computed in < 2 ms):                            │
  │                                                                       │
  │  U_charge  =  41.7%  PWM duty       ← MOSFET gate signal to STM32   │
  │  U_torque  =  0.68   scaling factor ← cap motor current 32% lower   │
  │  U_cool    =  88.0%  pump duty      ← force cooling at 88% power    │
  └──────────────────────────────────────────────────────────────────────┘

  WHY THESE SPECIFIC VALUES?
  The NPU has learned (from training) that when:
   • Temperatures are THIS high (119°C)
   • Current is THIS large (-12.257A discharge)
   • Risk score is THIS elevated (82.25/100)
  the optimal way to protect the pack without stopping the vehicle is
  to reduce charging to 41.7%, cap motor draw to 68%, and force cooling
  at 88% of maximum pump power.


================================================================================
  STEP 4 — ARDUINO UNO Q RECEIVES COMMANDS (via SRAM IPC)   ← 0.5 ms
================================================================================

  Snapdragon X writes 3 values to shared SRAM memory at 0x20000000:

     Byte 0x20000010 :  PWM_CHARGE  = 0x6A  (41.7% duty)
     Byte 0x20000011 :  PWM_MOTOR   = 0xAE  (68.0% scaling)
     Byte 0x20000012 :  PWM_COOL    = 0xE0  (88.0% pump)

  STM32 Zephyr task reads these within its next 10ms cycle:

  ┌──────────────────────────────────────────────────────────────────────┐
  │  Pin PB4  → MOSFET gate adjusted  → Charging current: 12A → 5.0A   │
  │  Pin PB5  → Motor controller      → Torque cap:  100% → 68%        │
  │  Pin PB8  → Coolant pump PWM      → Pump speed: 40% → 88%          │
  └──────────────────────────────────────────────────────────────────────┘
